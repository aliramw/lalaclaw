# Code of Conduct

## Our Standard

LalaClaw is an open-source project and we want participation to feel respectful, safe, and constructive for contributors across languages, experience levels, and backgrounds.

Examples of behavior that help create that environment include:

- showing empathy and patience in technical discussion
- giving actionable, respectful feedback
- assuming good intent while still being clear about problems
- welcoming contributors with different levels of experience
- focusing critique on ideas, code, and behavior rather than people

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, intimidation, or deliberate humiliation
- personal attacks, insults, or hostile language
- discriminatory language or conduct based on personal characteristics
- sexualized language or unwanted sexual attention
- doxxing, stalking, or publishing private information without permission
- sustained bad-faith arguing, trolling, or derailing contribution threads
- threatening language or encouraging harm toward others

## Scope

This Code of Conduct applies in project spaces, including:

- GitHub issues, pull requests, reviews, and discussions tied to this repository
- documentation and code review conversations
- community spaces or events that explicitly represent this project

## Reporting

If you experience or witness unacceptable behavior, please contact the maintainers privately rather than opening a public issue with sensitive details.

If you do not already have a private contact channel, use GitHub to contact the repository maintainers privately and request a conduct report path. For security-related problems, use the process documented in [SECURITY.md](./SECURITY.md).

## Enforcement

Maintainers may remove, edit, or reject comments, commits, code, issues, or other contributions that violate this Code of Conduct.

Depending on severity, maintainers may:

- request a change in behavior
- remove or lock content
- pause or restrict participation
- ban contributors from project spaces

## Project Commitment

Maintainers will aim to respond fairly, promptly, and with appropriate privacy for all parties involved.

This document is a lightweight project policy intended to keep collaboration healthy. It can be updated as the project grows and governance becomes more formal.
