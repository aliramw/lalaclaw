# Security Policy

## Supported Versions

Security fixes are only guaranteed for the latest state of the default branch.

## Reporting a Vulnerability

- Please do not open public GitHub issues for security-sensitive reports.
- Use GitHub Security Advisories when available, or contact the maintainer privately.
- Include reproduction steps, impact, and any suggested mitigation if you have one.

## Response Expectations

- Reports should be acknowledged within a few business days when possible.
- Follow-up questions may be needed to confirm scope and impact.
- Confirmed issues will be fixed and disclosed responsibly.

## High-Value Areas

- OpenClaw gateway credentials or token exposure
- Unsafe local transcript or attachment leakage
- File access bugs that escape the intended workspace
- Browser or terminal peek data exposure
